# 常见问题

## 安装和配置

### 插件支持哪些浏览器？

八爪鱼爬虫目前支持以下浏览器：
- Chrome 90+
- Microsoft Edge 90+
- Brave
- 其他基于 Chromium 的浏览器

**注意**：暂不支持 Firefox 和 Safari。

### 安装后找不到插件图标怎么办？

1. 打开浏览器扩展管理页面（chrome://extensions/）
2. 确认八爪鱼爬虫已启用
3. 点击扩展图标右侧的"固定"按钮，将图标固定到工具栏
4. 如果仍然看不到，尝试重启浏览器

### 插件需要哪些权限？

插件需要以下权限才能正常工作：
- **tabs**: 访问标签页信息
- **activeTab**: 与当前活动标签页交互
- **scripting**: 在网页中注入脚本
- **storage**: 保存采集规则和数据
- **alarms**: 定时任务调度
- **notifications**: 发送任务通知
- **host_permissions**: 访问网站内容

所有权限都用于数据采集功能，不会收集或上传您的个人信息。

## 数据采集

### 为什么采集不到数据？

可能的原因和解决方法：

**1. 选择器失效**
- 网站更新了页面结构
- 解决：重新选择元素，更新选择器

**2. 动态加载内容**
- 内容通过JavaScript动态加载
- 解决：添加等待时间或使用滚动触发

**3. 需要登录**
- 网站需要登录才能查看内容
- 解决：先在浏览器中登录，再执行采集

**4. 反爬虫限制**
- 网站检测到自动化访问
- 解决：降低采集频率，添加随机延迟

### 如何采集需要登录的网站？

方法一：手动登录
1. 在浏览器中正常登录网站
2. 保持登录状态
3. 使用插件进行采集

方法二：配置登录步骤
1. 在规则中添加登录步骤
2. 配置用户名和密码输入
3. 设置登录按钮点击
4. 等待登录完成后再采集

### 采集速度太慢怎么办？

优化建议：
1. **减少不必要的字段**：只采集真正需要的数据
2. **优化选择器**：使用更精确的CSS选择器
3. **调整并发数**：在设置中增加并发采集数量
4. **禁用图片加载**：如果不需要图片，可以禁用图片加载
5. **使用增量采集**：只采集新增或变更的数据

### 如何处理验证码？

目前插件不支持自动识别验证码。建议：
1. 先手动完成验证码验证
2. 在登录状态下进行采集
3. 使用API接口代替网页采集（如果网站提供）

## 规则配置

### CSS选择器和XPath有什么区别？

**CSS选择器**：
- 语法简洁，易于理解
- 浏览器原生支持，性能好
- 适合大多数场景
- 示例：`div.product > h2.title`

**XPath**：
- 功能更强大，支持复杂查询
- 可以向上查找父元素
- 支持文本内容匹配
- 示例：`//div[@class='product']/h2[@class='title']`

建议优先使用CSS选择器，复杂场景再考虑XPath。

### 如何选择稳定的选择器？

好的选择器应该：
1. **具有唯一性**：能准确定位到目标元素
2. **结构稳定**：不依赖容易变化的属性（如动态生成的class）
3. **层级适中**：不要太深也不要太浅
4. **语义明确**：使用有意义的class或id

推荐优先级：
1. `id` 属性（如果稳定）
2. 语义化的 `class` 名称
3. `data-*` 自定义属性
4. 结构选择器（如 `:nth-child()`）

### 如何处理动态内容？

对于通过JavaScript动态加载的内容：

**方法一：添加等待时间**
```
在步骤中添加"等待"动作
设置等待时间（如2秒）
```

**方法二：等待元素出现**
```
配置"等待元素"条件
指定要等待的元素选择器
设置最大等待时间
```

**方法三：滚动触发加载**
```
配置滚动动作
设置滚动距离或滚动到底部
等待内容加载完成
```

## 翻页和导航

### 支持哪些翻页方式？

插件支持以下翻页方式：

1. **点击下一页**：点击"下一页"按钮或页码链接
2. **无限滚动**：滚动到底部自动加载更多
3. **加载更多按钮**：点击"加载更多"按钮
4. **URL参数翻页**：修改URL中的页码参数

### 如何设置翻页次数限制？

在翻页配置中：
1. 找到"最大页数"设置
2. 输入要采集的最大页数
3. 或设置"最大数据量"限制
4. 达到限制后自动停止

### 翻页后数据重复怎么办？

启用去重功能：
1. 在数据保存设置中启用"去重"
2. 选择去重字段（如商品ID、URL等）
3. 设置去重策略（保留第一条或最新一条）

## 数据导出

### 支持哪些导出格式？

- **Excel (.xlsx)**：适合数据分析和报表
- **CSV (.csv)**：通用格式，可用于数据库导入
- **JSON (.json)**：适合程序处理和API对接
- **Google Sheets**：云端协作和实时同步

### 导出的Excel文件乱码怎么办？

这通常是编码问题：
1. 使用Excel打开CSV文件时选择UTF-8编码
2. 或者直接导出为Excel格式（.xlsx）
3. 如果仍有问题，尝试用记事本打开CSV，另存为UTF-8编码

### 如何导出到Google Sheets？

1. 在数据导出设置中选择"Google Sheets"
2. 点击"授权"按钮登录Google账户
3. 选择或创建目标工作表
4. 配置字段映射
5. 选择保存模式（追加/覆盖）
6. 开始导出

### 数据量很大时如何导出？

对于大量数据：
1. **分批导出**：按时间或分类分批导出
2. **使用CSV格式**：比Excel更适合大数据量
3. **直接保存到数据库**：配置数据库连接直接保存
4. **使用数据管道**：配置数据流处理管道

## 定时任务

### 如何设置定时任务？

1. 创建或编辑采集任务
2. 点击"定时设置"
3. 选择执行频率或输入Cron表达式
4. 启用定时任务
5. 保存设置

### Cron表达式怎么写？

Cron表达式格式：`分 时 日 月 周`

常用示例：
- `0 9 * * *` - 每天上午9点
- `0 */2 * * *` - 每2小时
- `0 9 * * 1-5` - 工作日上午9点
- `0 0 1 * *` - 每月1号零点
- `*/30 * * * *` - 每30分钟

可以使用在线Cron表达式生成器辅助编写。

### 定时任务没有执行怎么办？

检查以下几点：
1. 确认任务已启用
2. 检查Cron表达式是否正确
3. 确保浏览器在执行时间处于打开状态
4. 查看任务执行历史中的错误日志
5. 检查是否有权限限制

### 电脑关机后定时任务还会执行吗？

不会。定时任务需要：
1. 浏览器处于打开状态
2. 插件已启用
3. 电脑处于开机状态

如需24小时运行，建议：
- 使用云服务器部署
- 或使用我们的云端采集服务

## 性能和限制

### 一次可以采集多少数据？

理论上没有硬性限制，但建议：
- 单次任务不超过10000条数据
- 超过建议分批采集
- 注意浏览器内存占用

### 采集会被网站封禁吗？

可能会。为避免被封：
1. **控制采集频率**：设置合理的请求间隔
2. **模拟人工操作**：添加随机延迟
3. **使用代理IP**：轮换IP地址（高级功能）
4. **遵守robots.txt**：尊重网站的爬虫协议
5. **避免高峰期**：选择网站访问量低的时段

### 插件占用内存太大怎么办？

优化建议：
1. 减少并发采集数量
2. 及时清理历史数据
3. 关闭不需要的任务
4. 重启浏览器释放内存
5. 升级浏览器到最新版本

## 错误处理

### 常见错误代码

**ERR_SELECTOR_NOT_FOUND**
- 原因：找不到指定的元素
- 解决：检查选择器是否正确，页面是否已加载完成

**ERR_TIMEOUT**
- 原因：操作超时
- 解决：增加超时时间，检查网络连接

**ERR_NAVIGATION_FAILED**
- 原因：页面导航失败
- 解决：检查URL是否正确，网站是否可访问

**ERR_PERMISSION_DENIED**
- 原因：权限不足
- 解决：检查插件权限设置，重新安装插件

### 如何查看错误日志？

1. 打开插件主界面
2. 进入"任务管理"
3. 选择出错的任务
4. 点击"执行历史"
5. 查看详细错误信息

### 遇到问题如何反馈？

1. **查看文档**：先查看相关文档和FAQ
2. **搜索社区**：在用户社区搜索类似问题
3. **提交工单**：通过官网提交技术支持工单
4. **发送邮件**：发送详细问题描述到 support@dtiku.cn
5. **加入社群**：加入用户交流群获取帮助

反馈时请提供：
- 插件版本号
- 浏览器版本
- 目标网站URL（如果可以公开）
- 错误截图或日志
- 复现步骤
